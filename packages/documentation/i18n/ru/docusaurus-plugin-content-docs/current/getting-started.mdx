# Начало работы

Добро пожаловать в <CM/> 5 минутное руководство, где вы узнаете основы формата и его возможности! 

## Пример песни

Давайте рассмотрим пример песни, Hallelujah, Леонарда Коэна. 
Вы можете переключаться между исходным файлом, написанным в <CM/>, и отрендеренным вариантом, используя вкладки.

## Разбор по строкам

Давайте подробно разберем иллюстрированную выше иллюстрацию по строкам.

### Первый куплет

<TabbedChordMark src={data.sampleSong} srcLines={'1'} renderLines={'1'} />

Это метка раздела, которая определяет начало раздела: интро, куплет, припев...
`#` - это символ раздела; а `v` - сокращение для метки `Verse`.
<CM/> будет автоматически нумеровать разделы, используя тот же метки.

<TabbedChordMark src={data.sampleSong} srcLines={'2-3'} renderLines={'2-3'} />

Первая строка содержит два символа аккордов, каждый из которых сопровождается двумя точками. Это означает, что каждый аккорд должен звучать 2 такта.
Так как стандартный размер такта `4/4`, эта строка будет отображаться как один такт с 2 аккордами. 
Разделители тактов `|` добавляются автоматически.

Вторая строка содержит текст песни, исполняемый вместе с аккордами выше. Символ позиции `_` используется для указания точного места, где каждый аккорд должен звучать.

<TabbedChordMark src={data.sampleSong} srcLines={'4-5'} renderLines={'4-5'} />

Символ `%` инструктирует <CM/> повторять последнюю определённую строку аккордов, например, строку 2.

Хотя аккорды не определены явно, но повторяются, <CM/> все еще ожидает, что в строке текста песни должны присутствовать два аккорда. 

<TabbedChordMark src={data.sampleSong} srcLines={'6-7'} renderLines={'6-7'} />

Первая строка объявляет 2 такта с 2 аккордами каждый.
Так как в первом такте аккорды имеют неравномерную длительность (3 такта для `F`, 1 такт для `G`), индикатор счета такта `.` отображается явно. 
Он опускается, когда каждый аккорд в такте должен звучать в течение одинаковой длительности.

Вторая строка содержит только 3 маркера позиции для 4 аккордов, так как последнее изменение аккорда происходит после окончания текста песни.
В этом случае маркер позиции является необязательным, и <CM/> просто добавляет лишние аккорды в конце строки.

<TabbedChordMark src={data.sampleSong} srcLines={'12-13'} renderLines={'12-13'} />

Когда для аккорда не указана длительность, как для `Am` в этом втором такте, то <CM/> понимает, что он должен звучать в течение всего такта, например, 4 такта в этом случае.

### Первый припев

<TabbedChordMark src={data.sampleSong} srcLines={'15-23'} renderLines={'15-23'} />

`#c` объявляет новый раздел, используя сокращение `c` для метки `chorus`.
    
### Второй куплет

<TabbedChordMark src={data.sampleSong} srcLines={'25-31'} renderLines={'25-37'} />

Теперь это интересно: используя метку раздела `#v` второй раз, мы инструктируем <CM/> автоматически повторять аккорды, которые были определены для первого куплета, и применять их к текущему разделу.
Это означает, что в этом случае нужны только маркеры позиции аккордов, а не сами символы аккордов. 
Это огромная экономия времени при написании иллюстрированной иллюстрации!

Обратите внимание, что <CM/> автоматически нумерует второй куплет.

### Второй припев

<TabbedChordMark src={data.sampleSong} srcLines={'33'} renderLines={'39-47'} />

Так как второй припев идентичен первому, мы можем просто объявить метку и оставить содержимое раздела пустым: <CM/> будет понимать, что он должен автоматически повторить весь раздел.

Теперь предположим, что вы басист в группе и что вам важна только сетка аккордов.
