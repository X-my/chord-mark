# Разделы

Разделы лежат в основе некоторых из самых мощных функций, таких как автоматическое повторение аккордов на протяжении разделов одного типа.
Вы можете думать о них как о независимых частях песни, таких как Интро, Куплет, Припев, Соло...

Учитывая преимущества, которые они приносят, рекомендуется систематически использовать метки разделов.

## Объявление раздела

Чтобы создать новый раздел, используйте символ `#` за которым следует одно слово:

<TabbedChordMark src={'#intro\n#verse\n#bridge\n#chorus\n#verse\n#chorus'} />

Обратите внимание, что вы должны объявлять несколько разделов одной семьи с точно такими же метками раздела (например, `#verse`, `#chorus`...). <CM/> автоматически пронумерует их.

Метка раздела должна быть одним словом, состоящим только из букв, заглавных или строчных. 
Если присутствует другой символ, например, пробел, то строка будет рассматриваться как строка текста и раздел не будет объявлен.

<TabbedChordMark src={'Допустимые метки:\n#custom\n#mySection\n\nНедопустимые метки:\n#my section\n#1custom\n#custom 2'} />

Чтобы закончить раздел, просто объявите новый, или закончите файл:

<TabbedChordMark src={'#verse\nЭто куплет \n\n#chorus\nЭто припев'} />

## Сокращенная нотация

Чтобы ускорить процесс написания, <CM/> имеет сокращенную нотацию для наиболее часто используемых имен разделов. 
Вот список всех доступных сокращений. Рекомендуется использовать их:

<TabbedChordMark src={'#a\n#b\n#c\n#i\n#o\n#p\n#s\n#u\n#v'} />

## Повторение аккордов

Теперь пришло время к интересным моментам.
Как только вы укажете аккорды для раздела, они автоматически повторяются для следующих разделов, объявленных с точно такой же меткой.
Даже если строки аккордов больше не присутствуют, маркеры позиции аккордов могут быть определены точно так же, как если бы аккорды были явно написаны выше.

<TabbedChordMark src={data.repeat1} />

Если аккорды несколько отличаются в повторяемом разделе, просто переопределите их с новой строкой аккорда!
Переопределения применяются только к текущему повторяемому разделу:

<TabbedChordMark src={data.repeat2} />

## Повторение целого раздела

Если раздел повторяется с идентичными аккордами и текстом, просто напишите метку и оставьте тело раздела пустым. 

<TabbedChordMark src={data.repeat3} />

По умолчанию будет повторен первый объявленный раздел с этой меткой. Вы также можете целевым образом указать конкретный, добавив его индекс к метке:

<TabbedChordMark src={data.repeat4} />

## Умножение раздела

Если раздел должен быть сыгран несколько раз подряд, просто умножьте его:

<TabbedChordMark src={'#c x2\nE7 D7\nПрипев, сыгранный дважды подряд'} options={{ expandSectionMultiply: false }} />

Если рендерер <CM/> вызван с параметром `expandSectionMultiply: true`, умноженные разделы будут явно повторены.

<TabbedChordMark src={'#c x2\nE7 D7\nПрипев, сыгранный дважды подряд\n'} options={{ expandSectionMultiply: true }} />

Это работает и для повторяемых разделов тоже.
