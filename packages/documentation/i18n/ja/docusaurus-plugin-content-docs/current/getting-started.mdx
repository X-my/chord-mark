# はじめに

<CM/> 5分間のチュートリアルへようこそ！ここで、フォーマットの基本とその可能性を学びましょう！

## 例曲

例として、レナード・コーエンの「allelujah」を見てみましょう。
ソースファイル（<CM/>で書かれたもの）とレンダリングされたバージョンをタブで切り替えて、見比べることができます。

## 行ごとの解説

上記の和声譜を行ごとに詳しく解説します。

### 第一節のメイン

<TabbedChordMark src={data.sampleSong} srcLines={'1'} renderLines={'1'} />

これはセクションの開始を定義する部分タグです：前奏、メイン、副歌...
`#`はセクション記号；`v`は`Verse`の略記です。
<CM/>は同じタグを使用してセクションを自動的に番号付けします。

<TabbedChordMark src={data.sampleSong} srcLines={'2-3'} renderLines={'2-3'} />

第一行には二つの和声記号があり、それぞれの後ろに二つのドットがあります。これは各和声が二拍演奏されることを意味します。
デフォルトの拍子が`4/4`であるため、この行は二つの和声を持つ小節としてレンダリングされます。
小節区切り記号`|`は自動的に追加されます。

第二行には和声の上に歌詞があり、位置記号`_`は各和声がどこで演奏されるかを指定します。

<TabbedChordMark src={data.sampleSong} srcLines={'4-5'} renderLines={'4-5'} />

記号`%`は<CM/>が前行で定義された和声行を繰り返すことを指示します。例えば、第二行。

<CM/>は歌詞行に二つの和声があると期待します。

<TabbedChordMark src={data.sampleSong} srcLines={'6-7'} renderLines={'6-7'} />

第一行は二つの小節を宣言しています、それぞれに二つの和声があります。
最初の小節の和声の持続時間が均一でない（`F`が三拍、`G`が一拍）ため、拍子指示記号`.`が明示的にレンダリングされます。
各小節内の各和声が同じ持続時間で演奏される場合、それは省略されます。

第二行は三つの和声位置記号があり、四つの和声に対応しています最後の和声の変更が歌詞の終わりに発生する場合、その位置記号はオプショナルです。<CM/>は単に行末に余分な和声を追加します。

<TabbedChordMark src={data.sampleSong} srcLines={'12-13'} renderLines={'12-13'} />

和声に持続時間が指定されていない場合、ここでは第二小節の`Am`のように、<CM/>はそれが小節全体で演奏されることを理解します。例えば、ここでは四拍。

### 第一節の副歌

<TabbedChordMark src={data.sampleSong} srcLines={'15-23'} renderLines={'15-23'} />

`#c`は`c`の略記を使用して、新しいセクションを宣言します。

### 第二節のメイン

<TabbedChordMark src={data.sampleSong} srcLines={'25-31'} renderLines={'25-37'} />

今回は興味深い：第二回`#v`セクションタグを使用することで、<CM/>が最初の節で定義された和声を自動的に繰り返し、それを現在のセクションに適用することを指示します。
これは和声位置記号のみが必要で、和声記号自体は必要ないことを意味します。
これは和声譜を書くときに大量の時間を節約します！

<CM/>が第二節のメインを自動的に番号付けすることを注意してください。

### 第二節の副歌

<TabbedChordMark src={data.sampleSong} srcLines={'33'} renderLines={'39-47'} />

第二節の副歌が第一節の副歌と同じであるため、タグを宣言し、セクションの内容を空にしておくだけで済みます：<CM/>はそれが自動的に全セクションを繰り返すことを理解します。
