## 라인별 설명

위의 코드와 코드를 자세히 설명하겠습니다.

### 첫 번째 절 주요 가사

<TabbedChordMark src={data.sampleSong} srcLines={'1'} renderLines={'1'} />

이것은 섹션 레이블로, 시작 부분을 정의합니다: 전주, 주요 가사, 후렴...
`#`은 섹션 기호이며, `v`는 `Verse` 레이블의 약어입니다.
<CM/>은 동일한 레이블을 사용하여 섹션을 자동으로 번호 매깁니다.

<TabbedChordMark src={data.sampleSong} srcLines={'2-3'} renderLines={'2-3'} />

첫 번째 라인에는 두 개의 코드 심볼이 있으며, 각각 뒤에 두 개의 점이 있습니다. 이것은 각각 코드가 2박자를 연주해야 함을 의미합니다.
기본 박자는 `4/4`이므로, 이 라인은 2개의 코드가 있는 마디로 렌더링됩니다.
마디 구분자 `|`가 자동으로 추가됩니다.

두 번째 라인에는 코드 위에 가사가 있습니다. 위치 표시 기호 `_`는 각각 코드가 어디에서 연주해야 하는지를 지정합니다.

<TabbedChordMark src={data.sampleSong} srcLines={'4-5'} renderLines={'4-5'} />

`%` 기호는 <CM/>이 이전 라인에 정의한 코드 라인을 반복하도록 지시합니다. 예를 들어, 두 번째 라인입니다.

코드가 명확하게 정의되지 않았더라도, <CM/>은 여전히 가사 라인에 두 개의 코드가 있어야 한다고 예상합니다.

<TabbedChordMark src={data.sampleSong} srcLines={'6-7'} renderLines={'6-7'} />

첫 번째 라인은 두 개의 코드가 있는 두 개의 마디를 선언합니다.
첫 번째 마디의 코드 지속 시간이 불규칙하므로(`F`는 3박자, `G`는 1박자), 박자 지시 기호 `.`가 명시적으로 렌더링됩니다.
마디 내의 각각 코드가 동일한 지속 시간을 연주해야 할 때, 생략됩니다.

두 번째 라인은 세 개의 코드 위치 표시 기호만 포함하고 있습니다. 이는 4개의 코드에 해당합니다. 왜냐하면 마지막 코드 변경이 가사 끝에 발생하기 때문입니다.
이 경우, 위치 표시 기호는 선택 사항이며, <CM/>은 단순히 라인 끝에 추가 코드를 추가합니다.

<TabbedChordMark src={data.sampleSong} srcLines={'12-13'} renderLines={'12-13'} />

코드에 지속 시간이 지정되지 않은 경우, 예를 들어 두 번째 마디의 `Am`이 여기처럼, <CM/>은 그것이 전체 마디를 지속해야 한다고 이해합니다. 여기서는 4박자입니다.

### 첫 번째 절 후렴

<TabbedChordMark src={data.sampleSong} srcLines={'15-23'} renderLines={'15-23'} />

`#c`는 `c` 레이블의 약어를 사용하여 새로운 섹션을 선언합니다.

### 두 번째 절 주요 가사

<TabbedChordMark src={data.sampleSong} srcLines={'25-31'} renderLines={'25-37'} />

이제 재미있는 것이 있습니다: 두 번째로 `#v` 섹션 레이블을 사용하여, <CM/>에게 첫 번째 절에서 정의한 코드를 자동으로 반복하도록 지시하고, 현재 섹션에 적용하도록 지시합니다.
이것은 코드 라인에 위치 표시 기호만 필요하고, 코드 심볼 자체가 필요하지 않다는 것을 의미합니다.
이것은 코드를 작성할 때 많은 시간을 절약합니다!

<CM/>은 두 번째 절 주요 가사를 자동으로 번호 매깁니다.

### 두 번째 절 후렴

<TabbedChordMark src={data.sampleSong} srcLines={'33'} renderLines={'39-47'} />

두 번째 절 후렴이 첫 번째 절과 동일하기 때문에, 레이블을 선언하고 섹션 내용을 비워둡니다. <CM/>은 전체 섹션을 자동으로 반복하도록 이해합니다.
